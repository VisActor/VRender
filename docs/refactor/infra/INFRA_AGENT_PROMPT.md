- 严重影响开发效率和测试可靠性

## 🎯 你的任务

### 任务组：jest-electron 稳定性修复

#### 任务 1：调查崩溃根因
**必须完成的第一项**

**执行步骤**：
1. 查看崩溃日志（`Electron-2026-03-27-*.ips`）
2. 分析崩溃堆栈和触发条件
3. 检查当前配置：
   - Electron 版本
   - jest-electron 版本（现在是 `@pixi/jest-electron@26.1.0`）
   - jest 配置中的并行设置
4. 尝试复现：连续运行测试 5-10 次
5. 记录崩溃频率和触发条件

**输出**：
在 `INFRA_TASK_QUEUE.md` 中任务 1 下添加：
```markdown

- 根因：...
- 崩溃频率：...
- 触发条件：...
```

#### 任务 4：验证稳定性
**必须通过**

**验证方法**：
```bash
# 连续运行 10 次，统计崩溃次数
for i in {1..10}; do
  echo "Run $i"
  rush test -p @visactor/vrender-core 2>&1 | tee test-run-$i.log
done
```

**通过标准**：
- 10 次运行 0 崩溃
- 或崩溃率 < 5%（如果无法完全消除）

**输出**：
在 `INFRA_TASK_QUEUE.md` 中任务 4 下添加：
