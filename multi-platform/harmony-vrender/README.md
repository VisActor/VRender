# VRender Harmony Smoke

This directory is a HarmonyOS host project for VRender multi-platform smoke and capability checks.

Open this directory in DevEco Studio, not the repository root. The root repository is a Rush monorepo and is not a Harmony project.

## Local Environment

The current verified setup on macOS uses DevEco Studio with command line tools from the app bundle:

```sh
export PATH="/Applications/DevEco-Studio.app/Contents/tools/ohpm/bin:/Applications/DevEco-Studio.app/Contents/sdk/default/openharmony/toolchains:/Applications/DevEco-Studio.app/Contents/tools/hvigor/bin:$PATH"
export DEVECO_SDK_HOME="/Applications/DevEco-Studio.app/Contents/sdk"
```

Check the tools:

```sh
ohpm -v
hdc list targets
hvigorw --version
```

On the initial local setup, `ohpm -v` returned `6.1.1.830` and `hdc list targets` showed `127.0.0.1:5555`.

## Build

From this directory:

```sh
node scripts/sync-local-vrender.cjs
```

This creates `entry/src/main/ets/vendor/vrender/index.js` from the current VRender source tree. The generated bundle is ignored by git and should be refreshed after VRender runtime changes.
The sync script also applies the build-time constants normally injected by the VRender package build, including `__VERSION__` and `__DEV__`.

```sh
hvigorw assembleHap --mode module -p module=entry@default -p product=default
```

DevEco may warn that no signing profile is configured. That is expected for this copied smoke project and does not block command line build output. Configure signing in DevEco before installing a HAP from the command line.

If DevEco or hvigor warns that project dependencies are not installed, run `ohpm install --all`. The generated `oh_modules/` and `oh-package-lock.json5` remain local files and should not be committed.

If the app exits immediately with `ReferenceError: __VERSION__ is not defined` from `entry/src/main/ets/vendor/vrender/index.js`, the local VRender bundle was generated by an outdated sync path. Re-run `node scripts/sync-local-vrender.cjs`, then rebuild or rerun from DevEco Studio.

## Manual Run

Run the `entry` module from DevEco Studio on the Harmony simulator. The page should show two canvases:

- the upper native Harmony Canvas baseline
- the lower VRender Stage rendered through `envParams.canvasFactory`

Expected console logs use the `[harmony-vrender]` prefix. Touching the lower VRender canvas should update the on-screen event line and call `stage.window.dispatchEvent`; touching the blue rect should toggle its fill color if VRender picking is active on the simulator.

The VRender section is organized as a multi-scene smoke page. Use the scene buttons to switch coverage and use the action buttons below the VRender canvas:

- `主操作`: the primary update for the current scene.
- `次操作`: the secondary update for scenes that need a second control path.
- `重绘`: redraws the current scene.
- `重建Stage`: releases and recreates the VRender Stage while keeping the shared app path.
- `释放Stage`: releases only the Stage; use `重建Stage` or `重绘` to create it again.

### Manual Scene Checklist

- `图元`: rect/circle/symbol/line/area/arc/path/polygon render with gradient, shadow, opacity, lineDash and corner radius. Tap rect/circle and run both actions.
- `文本`: normal text, multiline text, wrap text, rich text, underline and lineThrough render without overlap.
- `资源`: records current image/canvas-resource capability. If Harmony does not expose a usable image resource path, the scene must show a clear fallback instead of crashing.
- `动画`: loop, bounce, from, wait, color/position/size/opacity animation. `主操作` pauses/resumes the moving rect; `次操作` stops the orange rect with `stop(end)`.
- `状态`: setStates, same-state patch, addState/removeState and toggleState. Tap the diamond and run both actions.
- `变换`: nested group, clip, opacity, zIndex, rotation/scale/translate. Tap the transformed group and run `主操作`.
- `组件`: Tag, Segment, ArcSegment, LineAxis, CircleAxis, DiscreteLegend, Slider, DataZoom, ScrollBar, CheckBox, Radio, Switch, Title, Indicator and Tooltip. Components that fail on Harmony should render a small `failed` fallback and log the error.
- `事件`: overlapping graphics, zIndex, rotated group and clipped children. Tap each target and check the on-screen event line.
- `批量`: 48 graphics with batch `setAttribute`; tap random nodes and run `主操作`.
- `几何`: line/area/polygon/path/arc/symbol geometry updates. Run `主操作` repeatedly.
- `生命周期`: remove/recreate child nodes, scene switch cleanup and Stage release/recreate. Use this together with `重建Stage` and `释放Stage`.

## Current Smoke Scope

`entry/src/main/ets/pages/Index.ets` currently verifies the Harmony host itself:

- ArkUI `Canvas` creation
- native `CanvasRenderingContext2D` drawing
- fill, stroke, arc, path, gradient and line dash
- touch down, move, up and cancel event delivery
- console logging with the `[harmony-vrender]` prefix
- VRender `acquireSharedVRenderApp({ env: 'harmony' })`
- VRender Stage creation through `envParams.canvasFactory`
- VRender multi-scene smoke rendering on a Harmony `Canvas`
- touch forwarding to `stage.window.dispatchEvent`

The native canvas baseline and the VRender stage are intentionally shown separately. This keeps Harmony host capability issues distinct from VRender env registration, stage lifecycle, and event forwarding issues.

## Repository Hygiene

Do not commit generated or local-machine files:

- `.idea/`
- `.hvigor/`
- `oh_modules/`
- `build/` and `entry/build/`
- `local.properties`
- `oh-package-lock.json5`

These paths are ignored by the copied Harmony project.
