# 总监与协调者 - 工作指南

> **你的身份**：总监与协调者
> **你的职责**：制定目标、协调资源、监控进度、审查决策
> **团队**：架构设计师（Claude）+ 资深一线开发者（另一个 Agent）

---

## 👥 团队角色

### 总监与协调者（你）
- **职责**：制定战略目标、协调多个 AI、审查关键决策
- **工作**：监控进度、处理跨 AI 冲突、最终验收

### 架构设计师（Claude）
- **职责**：架构设计、任务规划、质量审查
- **输出**：`TASK_PLAN.md`、`TASK_QUEUE.md`、架构设计文档
- **不负责**：具体代码实现

### 资深一线开发者（另一个 Agent）
- **职责**：执行任务、编写代码、标记完成状态
- **输出**：测试代码、实现代码、完成标记
- **遵循**：`TASK_TEMPLATE.md` 规范

## 🚀 如何开始

### 步骤 1：启动资深一线开发者
将以下内容提供给另一个 Agent：

```
你的身份是资深一线开发者，负责为 VRender 项目补充关键测试。

请按顺序阅读以下文档：
1. AGENT_PROMPT.md - 完整的任务执行指南（重要！）
2. TASK_QUEUE.md - 任务队列
3. TASK_TEMPLATE.md - 测试编写规范

你的任务是从 TASK_QUEUE.md 中选择未完成的任务（优先 P0），编写测试代码，并在文档中标记完成状态。

推荐第一个任务：任务 1.1 - useStates 核心行为测试
```

### 步骤 2：监控进度
定期查看 `TASK_QUEUE.md` 的进度：
```bash
# 查看进度
grep -A 5 "### 统计信息" TASK_QUEUE.md

# 查看已完成任务
grep "✅ 已完成" TASK_QUEUE.md

# 查看阻塞任务
grep "🚫 阻塞" TASK_QUEUE.md
```

### 步骤 3：处理问题
如果发现：
- 🚫 阻塞任务：评估是否需要调整优先级或提供资源
- ❓ 需求问题：与架构设计师讨论后决策
- ⚠️ 架构问题：要求架构设计师重新设计

## 🎯 决策点

### 需要你决策的情况
1. **任务优先级调整**：是否允许跳过某个 P0 任务
2. **架构问题处理**：发现设计缺陷时的处理方案
3. **资源分配**：是否需要增加或减少 Agent 资源
4. **时间调整**：是否延长或缩短准备阶段时间

### 你不需要决策的情况
- 测试用例的具体编写方式（资深一线开发者自主决策）
- 测试数据的选择（资深一线开发者自主决策）
- 小的 bug 发现（记录即可，不需要立即修复）
